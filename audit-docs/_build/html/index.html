Biosample audits¶

igvfd.audit.biosample.audit_biosample_nih_institutional_certification(value, system)¶

Check if Biosample objects specify an NIH Institutional Certification which is required for human data.

The function is meant to audit the ‘Biosample’ objects in the system, verifying that each Biosample intended for human data has an associated NIH Institutional Certification. If a Biosample object is associated with a human donor but does not specify a NIH Institutional Certification, it raises an audit failure.

Parameters:

  • value (dict) – The Biosample object to be audited.

  • system (any) – System specific parameters.

Yields:

AuditFailure – An AuditFailure object specifying the Biosample object that fails the check.

igvfd.audit.biosample.audit_biosample_taxa_check(value, system)¶

Audit function to flag biosamples associated with donors of different taxa.

This function audits ‘Biosample’ objects in the system to verify that all donors associated with a Biosample object belong to the same taxa. If a Biosample is associated with donors from more than one taxa, it raises an audit failure.

Parameters:

  • value (dict) – The Biosample object to be audited.

  • system (any) – System specific parameters.

Yields:

AuditFailure – An AuditFailure object specifying the Biosample object that fails the check.

Construct library audits¶

igvfd.audit.construct_library.audit_construct_library_associated_diseases(value, system)¶

Audit ConstructLibrary objects for correct associated diseases specification.

This function checks if ConstructLibrary objects with origins of disease-associated variants specify which associated_disease is relevant, and vice versa. If inconsistency is found, it raises an audit failure.

Parameters:

  • value (dict) – The ConstructLibrary object to be audited.

  • system (any) – System specific parameters.

Yields:

AuditFailure – An AuditFailure object specifying the ConstructLibrary object that fails the check.

igvfd.audit.construct_library.audit_construct_library_plasmid_map(value, system)¶

Audit ConstructLibrary objects for plasmid map document attachment.

This function verifies that ConstructLibrary objects have a document of type “plasmid map” attached to them. If such a document is missing, the function raises an audit failure.

Parameters:

  • value (dict) – The ConstructLibrary object to be audited.

  • system (any) – System specific parameters.

Yields:

AuditFailure – An AuditFailure object specifying the ConstructLibrary object that fails the check.

Human donor audits¶

igvfd.audit.human_donor.audit_related_donors(value, system)¶

Audit HumanDonor objects for unique and mutually specified related donors.

This function checks whether each HumanDonor has unique related donors and if the related donors are mutually specified between each other. If any inconsistencies are found, it raises an audit failure.

Audit level:

INTERNAL_ACTION

Parameters:

  • value (dict) – The HumanDonor object to be audited.

  • system (any) – System specific parameters.

Yields:

AuditFailure – An AuditFailure object specifying the HumanDonor object that fails the check.
