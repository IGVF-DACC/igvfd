Schema Overview

10 concrete file schemas all inherit from the abstract file.json base. Common required fields: award, lab, md5sum, file_format, content_type.

Base schema (file.json) defines common properties: accession, analysis_step_version, content_md5sum, derived_from, derived_manually, file_format, file_format_specifications, file_set, file_size, md5sum, submitted_file_name, upload_status, validation_error_detail, supersedes. Subclasses override file_format and content_type with concrete enums.

Inheritance Hierarchy

All file schemas inherit from the abstract file.json base via mixinProperties. Each subclass overrides content_type and file_format with concrete enumerations.

file.json Abstract base — award, lab, md5sum, file_format, content_type
Sequence File bam · fastq · pod5
Alignment File bam · cram
Signal File bigWig · npz
Index File bai · crai · tbi
Tabular File tsv · csv · bed · vcf · +5
Reference File fasta · gtf · vcf · bed · +11
Model File hdf5 · json · pkl · pt · +3
Configuration File yaml · json · tsv
Image File jpg · png · svg · tar
Matrix File h5ad · mtx · hic · cool · +7
Mixin inheritance pattern: Each subschema lists file.json#/properties as its first mixinProperties entry, gaining all base properties. Then it adds its own mixins (e.g., controlled_access, filtered, base_modifications). dependentSchemas are merged from file.json#/dependentSchemas using the $merge directive.

Validation Constraints & Rules

Each schema defines conditional validation rules using dependentSchemas. All schemas inherit the base file.json constraints.

● file.json (Base — inherited by all)

upload_status → file_size required
When upload_status = "validated", file_size must be present.
status → upload_status restriction
Released/archived/revoked files cannot have upload_status of "pending" or "file not found". Also requires release_timestamp.
derived_manually → analysis_step_version
If derived_manually = true, analysis_step_version must NOT be present.
file_format → assembly required
bed, bedpe, bigBed, bigInteract, gvcf, vcf formats require submitted_assembly, assembly, or reference_files.
transcriptome_annotation ↔ assembly consistency
GRCh38 → GENCODE 22–47. GRCm39 → GENCODE M30–M36. Custom assembly → no annotation allowed.

● Sequence File

fastq + validated → read statistics required
Validated FASTQ files must have minimum_read_length, maximum_read_length, mean_read_length, and read_count.
content_type ↔ file_format pairing
"reads" → fastq only. "PacBio subreads" / "PacBio consensus reads" → bam only. "Nanopore reads" → pod5 only.
base_modifications → Nanopore reads only
Base modification data can only be specified for content_type = "Nanopore reads".

● Alignment File

content_type → reference_files dependency
"alignments", "alignments with modifications", and "transcriptome alignments" require reference_files. "methylated reads" does NOT require it.
base_modifications → modification content types
Can only specify base modifications for "alignments with modifications" or "methylated reads" content types.

● Signal File

content_type ↔ file_format pairing
"sequence attributes" must use npz format. All other content types are restricted to bigWig.

● Index File

file_format → derived_from type restriction
bai/crai index files must be derived from an AlignmentFile. tbi index files must be derived from a ReferenceFile or TabularFile.
derived_from maxItems: 1
Index files can only be derived from exactly one source file (unlike other schema types).

● Tabular File

bed/bigBed → file_format_type required
bed and bigBed formats must specify a file_format_type. Other formats (tsv, csv, etc.) must NOT have it.
~75 content types → reference context required
Most genomic content types require reference_files, submitted_assembly, or submitted_transcriptome_annotation.
base_modifications → modification state types only
Only "modification state", "minus strand modification state", and "plus strand modification state" can have base modification data.

● Reference File

genome reference/index → assembly required
"genome reference" and "genome index" content types require assembly.
transcriptome reference → assembly + annotation
Non-custom assemblies require both assembly and transcriptome_annotation. Custom assemblies only need assembly.
bed → file_format_type required
bed format requires file_format_type; vcf and all other formats must NOT have it.

● Model File

content_type ↔ file_format pairing
"graph structure" → json only. "edge weights" → hdf5 only. All other model content types are format-agnostic.

● Configuration File

seqspec → yaml format only
content_type = "seqspec" is restricted to file_format = "yaml".
seqspec → seqspec_of allowed
Only seqspec configuration files may use the seqspec_of property to link to sequence files.
